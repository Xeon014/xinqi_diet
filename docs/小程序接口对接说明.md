# 小程序接口对接说明

## 基础约定

- 所有接口统一使用 `/api` 作为前缀。
- 所有响应统一使用 `ApiResponse<T>`。
- 小程序侧优先判断 HTTP 状态码，其次判断 `code` 是否为 `SUCCESS`。
- 除登录接口外，业务接口统一携带 `Authorization: Bearer {accessToken}`。

统一响应结构：

```json
{
  "code": "SUCCESS",
  "message": "ok",
  "data": {},
  "timestamp": "2026-03-07T23:40:00"
}
```

```json

  "code": "wx.login返回的code",
  "clientUserKey": "客户端本地用户键(开发 mock 可选)"
}
```

关键返回字段：

- `isNewUser`

`PUT /api/users/{userId}`

## 小程序接入建议

- `app.onLaunch` 先执行 `wx.login -> /api/auth/wechat/login`，保存 `accessToken + userId`。
- 请求层统一自动附带 `Authorization`，401 时自动重登一次。
- 首页调用 `/api/users/{id}/daily-summary` 展示饮食摄入、运动消耗和净摄入。
- 首页记录链路分两条入口：饮食记录（`/api/records`）与运动记录（`/api/exercise-records`）。
- 首页左滑“再记一次”不再直接创建记录，而是先通过 `/api/records/{id}` 或 `/api/exercise-records/{id}` 拉取详情，进入预填新增态。
- 食物/运动选择页右上角“历史记录”入口分别调用 `/api/records/history`、`/api/exercise-records/history`。
- 趋势页调用 `/api/body-metrics/snapshot` 和 `/api/body-metrics/trend` 展示体征变化。
- 首页在记录区下方展示”健康日记”入口与当日摘要，点击进入独立编辑页。

## 数据导入

### 40. 预览体重数据导入

`POST /api/body-metrics/import/preview`

说明：

- 接收 CSV / XLSX 文件内容，解析后返回预览数据，**不写入数据库**。
- 自动检测文件类型、分隔符（CSV）、日期列和体重列、日期格式、单位（kg/斤/lbs）。
- 支持带时间的日期文本，解析后仅保留日期部分，例如 `2026/3/28 09:39` 最终按 `2026-03-28` 处理。
- 单位从体重列名中提取（如 `体重 (斤)` → 斤，`Weight (lbs)` → lbs），并自动转换为 kg。

请求：

```json
{
  “fileName”: “export.xlsx”,
  “fileBase64”: “base64-encoded-file-content”
}
```

关键返回字段：

- `totalRows` — 数据行总数（不含表头）
- `parsedRows` — 成功解析的行数
- `skippedRows` — 无法解析的行数
- `detectedFileType` — 检测到的文件类型（`CSV` / `XLSX`）
- `detectedSheetName` — 检测到的工作表名称（仅 XLSX 返回）
- `detectedDelimiter` — 检测到的分隔符（`,` / `;` / `Tab`）
- `detectedUnit` — 检测到的单位（`kg` / `斤` / `lbs`）
- `rows[].rawDate` — 原始日期文本
- `rows[].rawWeight` — 原始体重文本
- `rows[].parsedDate` — 解析后的日期（解析失败为 null）
- `rows[].parsedWeightKg` — 解析后的体重 kg（解析失败为 null）
- `rows[].error` — 错误信息（无错误为 null）

当前小程序调用方：

- `pages/weight-import`

### 37. 确认体重数据导入

`POST /api/body-metrics/import/confirm`

说明：

- 将预览确认的数据写入数据库。
- `duplicatePolicy` 为 `SKIP` 时跳过已有记录，为 `OVERWRITE` 时覆盖已有记录。
- 如果导入数据包含今日日期，会同步更新用户当前体重和目标方案。
- 单次最多 1000 条。

请求：

```json
{
  “rows”: [
    { “date”: “2025-01-01”, “weightKg”: 70.5 },
    { “date”: “2025-01-02”, “weightKg”: 71.0 }
  ],
  “duplicatePolicy”: “SKIP”
}
```

关键返回字段：

- `imported` — 成功导入数
- `skipped` — 跳过数
- `overwritten` — 覆盖数
- `errors` — 错误信息列表

当前小程序调用方：

- `pages/weight-import`
