- 变更范围:
  - 页面/模块: `backend/src/main/java/com/diet/api/food/*`, `backend/src/main/java/com/diet/app/food/*`, `backend/src/main/java/com/diet/infra/food/*`, `miniprogram/services/food.js`

  - 同步更新接口文档、后端说明和云托管部署清单。
- 为什么做:
  - 先把核心技术链路打通，验证“营养成分表拍照识别 -> 生成自定义食物草稿”在当前架构下可落地。
- 验证方式:
  - 命令: `cd backend && mvn -q test -Dtest=NutritionLabelParserTest,NutritionLabelRecognitionServiceTest`
  - 手动回归页面: 无，本次仅补后端能力与小程序服务层
- 风险与回滚点:
  - 风险主要在腾讯云 OCR 的真实响应格式与本地签名调用；如需回滚，可删除新增识别接口与相关服务，不影响现有记录链路。
- 后续待办:
  - 接入实际页面拍照入口与图片 Base64/上传策略。
  - 基于真实样本继续收敛营养字段解析规则与异常值判定。
