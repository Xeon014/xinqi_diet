# 当前体验问题清单（初稿）

> 目标：把“体验不达标点”结构化沉淀，便于逐条收敛。

## 记录页

### UX-R-01 日期条可点击性与信息层级
- 问题：日期切换与日期选择器在小屏机型可能显得拥挤。
- 影响：用户切换日期时操作成本上升。
- 目标状态：一眼看出“当前日期 + 左右切换 + 选择器”三种操作，触控区充足。
- 验收口径：主流机型下按钮不重叠、文本不截断、误触率可接受。

### UX-R-02 摘要卡阅读优先级
- 问题：已摄入/已消耗/目标/剩余四项在视觉上权重接近。
- 影响：用户可能难以第一眼判断今日核心状态。
- 目标状态：关键状态（剩余/超出）更突出，次级指标权重降低。
- 验收口径：用户 3 秒内可回答“今天还差多少/超了多少”。

## 首页

### UX-H-01 总览卡信息密度
- 问题：数值信息已精简，但重点数字与说明文案还可再拉开层级。
- 影响：首屏“状态判断”速度可能不稳定。
- 目标状态：主数字最突出，状态标签次之，解释文案最轻。
- 验收口径：用户无滚动即可完成“状态感知 + 下一步操作”。

### UX-H-02 主按钮语义稳定性
- 问题：推荐餐次在跨时段切换时，文案变化可能引起认知跳变。
- 影响：用户预期与按钮结果偶尔不一致。
- 目标状态：按钮语义稳定且可预期，必要时增加轻提示。
- 验收口径：在早/午/晚/夜四时段测试下均符合预期。

### UX-H-03 当日状态标签策略（后期优化项）
- 问题：固定“当日状态”标签在首页价值不稳定，容易形成视觉噪声。
- 影响：占用主卡头部空间，弱化“净热量”主信息的聚焦。
- 目标状态：后续如需恢复状态标签，必须采用动态状态映射（如超标/平衡/待补记录）并提供明确判定规则。
- 验收口径：标签恢复后，用户能在 3 秒内理解其语义且不干扰主数字阅读。

## 我的页

### UX-M-01 头部信息可读性
- 问题：头部摘要在不同用户名长度下可能出现排版挤压。
- 影响：美观与可读性下降。
- 目标状态：长短用户名都保持稳定布局。
- 验收口径：极短名/超长名下无重叠与截断错位。

### UX-M-02 工具区信息精炼
- 问题：工具描述文本偏长时会带来视觉噪声。
- 影响：列表观感不够“轻”。
- 目标状态：标题清晰、描述短句化，整体节奏统一。
- 验收口径：三项工具在同屏下节奏一致，不显拥挤。

## 套餐管理

### UX-C-01 编辑态步骤负担
- 问题：编辑态功能完整，但新用户首次使用学习成本偏高。
- 影响：可能出现“会看不会改”的情况。
- 目标状态：编辑流程更线性，关键操作更明确。
- 验收口径：首次用户在 1 分钟内完成“改名 + 改克重 + 保存”。

## 趋势页

### UX-T-01 入口缺失
- 问题：趋势页存在但无可见入口。
- 影响：用户无法触达统计能力。
- 目标状态：在首页或我的页提供清晰入口。
- 验收口径：用户 2 步内可进入趋势页。

## 使用建议

- 先按高频路径优先：记录页 -> 首页 -> 我的页 -> 套餐页 -> 趋势页。
- 每次只处理 1~2 条问题，改完即回归验证并更新本文件状态。
