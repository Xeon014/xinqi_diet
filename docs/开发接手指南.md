# 开发接手指南

> 目标：让新的开发者或 AI agent 在另一台机器上接手本项目时，尽快完成启动、定位、验证和文档同步。

## 1. 环境要求

- Java 17
- Maven 3.9+
- MySQL 5.7+，并始终按 MySQL 5.7 兼容口径开发
- Node.js（用于小程序脚本校验）
- 微信开发者工具（小程序调试）

## 2. 首次阅读顺序

1. `README.md`
2. `docs/面向AI Agent的项目总览.md`
3. `backend/README.md`
4. `miniprogram/README.md`
5. `docs/小程序接口对接说明.md`
6. `AGENTS.md`

## 3. 仓库准备

1. 拉取代码并进入项目根目录
2. 确认关键目录存在：
   - `backend/src/main/java`
   - `backend/src/main/resources`
   - `miniprogram/`
   - `docs/`
3. 如果接手的是未完成分支，先执行 `git status`，确认当前工作区是否已有未提交改动

## 4. 后端启动

1. 进入目录：`cd backend`
2. 如需初始化数据库账号与权限，执行：`scripts/mysql-init.sql`
3. 如需从空库或删库后重建最新结构，执行：`scripts/sql/schema-full-latest.sql`
4. 如需初始化内置食物，执行：`scripts/sql/bootstrap-builtin-foods-latest.sql`
5. 如需初始化内置运动，执行：`scripts/sql/bootstrap-builtin-exercises-latest.sql`
6. 启动开发环境：`mvn spring-boot:run`
7. 编译检查：`mvn -q -DskipTests compile`

补充说明：

- 结构维护规则以 `backend/src/main/resources/db/migration/`、`backend/src/main/resources/schema.sql` 和 `backend/scripts/sql/schema-full-latest.sql` 为准
- 运行与部署说明优先看 `backend/README.md` 和 `backend/docs/`

## 5. 小程序启动

1. 用微信开发者工具打开 `miniprogram/`
2. 检查运行时配置入口：`miniprogram/utils/constants.js`
3. `develop` 环境默认直连本地后端 `127.0.0.1:8080`
4. 真机调试时，把后端地址改为局域网可访问 IP，不要继续使用 `127.0.0.1`

## 6. 常用源码入口

- 后端 HTTP 入口：`backend/src/main/java/com/diet/trigger/controller`
- 后端应用服务：`backend/src/main/java/com/diet/app`
- 后端接口 DTO：`backend/src/main/java/com/diet/api`
- 后端 MyBatis XML：`backend/src/main/resources/mapper`
- 小程序页面：`miniprogram/pages`
- 小程序请求层：`miniprogram/services`
- 小程序全局配置：`miniprogram/utils/constants.js`
- 趋势页实现：`miniprogram/pages/progress`
- 仓库级文档入口：`docs/面向AI Agent的项目总览.md`

## 7. 最低验证动作

- 仅改小程序页面、组件、服务：
  - `cd miniprogram && node scripts/verify-miniprogram-files.js`
- 改后端 Java 代码：
  - `cd backend && mvn -q -DskipTests compile`
- 改高风险后端逻辑或接口：
  - `cd backend && mvn -q test`
- 改跨应用接口：
  - 后端编译/测试
  - 小程序校验
  - 手动回归对应页面

## 8. 文档同步清单

- 页面入口或跳转参数变化：
  - `miniprogram/docs/页面入口与跳转地图.md`
- 功能状态或当前交互口径变化：
  - `miniprogram/docs/功能总览清单.md`
- 接口字段、参数名、返回结构变化：
  - `docs/小程序接口对接说明.md`
- 跨应用认知、阅读顺序、任务入口变化：
  - `README.md`
  - `docs/README.md`
  - `docs/面向AI Agent的项目总览.md`
- 每轮迭代结束：
  - `docs/变更日志.md`

## 9. 接手时最容易踩错的点

- 数据库必须按 MySQL 5.7 兼容思路开发，不要引入 MySQL 8 专属能力
- 表结构变更必须走 Flyway migration，不能只改运行时代码
- 小程序接口改动要同时检查 `miniprogram/services/` 与页面调用
- 中文文档和说明文件优先保持 UTF-8，避免在 Windows 终端批量写入时出现乱码
- 趋势页、首页记录链路、体重导入是当前高频协作区域，修改后一定回写文档
