# 开发接手指南（初稿）

> 适用于在另一台电脑继续本项目开发。

## 1. 环境要求

- Java 17
- Maven 3.9+
- MySQL 8+
- Node.js（用于小程序脚本校验）
- 微信开发者工具（小程序调试）

## 2. 仓库准备

1. 拉取代码并进入项目根目录。
2. 检查关键目录：
   - `backend/src/main/java`（后端）
   - `backend/src/main/resources`（配置与 SQL）
   - `miniprogram/`（小程序）
   - `docs/`（文档）

## 3. 后端启动

1. 进入后端目录：`cd backend`
2. 初始化数据库账号与权限（如需要）：`scripts/mysql-init.sql`
3. 从空库或删库后重建最新结构：`scripts/sql/schema-full-latest.sql`
4. 手工导入内置食物：`scripts/sql/bootstrap-builtin-foods-latest.sql`
5. 手工导入内置运动：`scripts/sql/bootstrap-builtin-exercises-latest.sql`
6. 结构维护规则参见：`backend/docs/数据库删库重建与 Flyway 基线.md`
7. 开发环境启动：
   - `mvn spring-boot:run`
8. 编译检查：
   - `mvn -q -DskipTests compile`

## 4. 小程序启动

1. 用微信开发者工具打开 `miniprogram/`。
2. 检查请求地址配置：
   - `miniprogram/utils/constants.js`
3. 若真机调试，确保后端地址不是 `127.0.0.1`，改为局域网可访问 IP。

## 5. 必跑校验

- 每次修改小程序后执行：
  - `cd miniprogram && node scripts/verify-miniprogram-files.js`
- 若校验失败，不要继续开发新功能，先修复编码/JSON问题。

## 6. 编码与文件注意事项

- 小程序文件（`js/wxml/wxss`）使用 UTF-8（无 BOM）。
- Java 源码使用 UTF-8（无 BOM），避免 `\ufeff` 编译错误。
- 中文文案保持中文，不用英文占位替代。

## 7. 推荐开发流程（高稳定）

1. 先看 `miniprogram/docs/功能总览清单.md`，确认目标功能编号。
2. 只改一个小功能。
3. 跑小程序校验 + 后端编译。
4. 手动回归相关页面。
5. 更新 `docs/变更日志.md`。

## 8. 常见问题速查

- 小程序编译报 WXML 条件渲染错误：检查 `wx:if/wx:elif/wx:else` 链式写法。
- Java 编译报非法字符 `\ufeff`：检查文件 BOM，转 UTF-8 无 BOM。
- 页面中文乱码：优先检查文件编码与终端写入方式。

## 9. 前端交互术语速记

### 9.1 词云

- 不是“云计算”的云，这里指一组关键词标签密集铺开显示。
- 常见表现：每个词像一个小胶囊标签，长度不一，自动换行排列。
- 典型用途：快速展示“最近搜索”“热门标签”“兴趣标签”这类轻量信息。
- 在本项目里的接近例子：食物/运动选择页的“最近搜索”标签区。
- 为什么通常不适合左滑删除：
  - 单个标签面积小，横向滑动手势不稳定。
  - 标签会换行，不是规整的单列列表。
  - 用户更容易理解“点一下使用”“点叉删除”，不容易理解“左滑删除”。

### 9.2 网格

- 指规则排列的多行多列布局，单元格大小通常比较接近。
- 可以把它理解成“九宫格/相册格子/商品宫格”这一类结构。
- 典型用途：图片、商品、功能入口、头像等并列展示。
- 在本项目里的接近例子：健康日记的图片区域。
- 为什么通常不适合左滑删除：
  - 用户操作方向不固定，容易和页面滚动、图片预览冲突。
  - 单元格本来就小，再叠加滑动手势，误触概率会升高。

### 9.3 卡片矩阵

- 可以理解成“由多个信息卡片组成的二维排布”。
- 它和网格的区别在于：网格更强调整齐、轻量；卡片矩阵里的每个块通常信息更多，层级更丰富，可能还有按钮或状态。
- 典型用途：仪表盘、概览页、统计页、能力入口页。
- 在本项目里的接近例子：趋势页指标卡、我的页工具卡。
- 为什么通常不适合左滑删除：
  - 卡片本身已经承载了点击、查看、编辑等主操作。
  - 左滑会和卡片内部点击区域、按钮区域产生冲突。
  - 用户会把卡片当“信息块”而不是“列表项”，对左滑预期更弱。

### 9.4 和“单列列表”的区别

- 单列列表：一行一个对象，自上而下排列，最适合放左滑删除。
- 词云：很多小标签自动换行，不是按“行项”理解。
- 网格：多行多列，强调并列展示。
- 卡片矩阵：也是多块并列，但每块信息更重、更像小面板。

### 9.5 一个实用判断方法

- 如果用户看到页面后，会自然把内容理解成“一条一条记录”，通常就适合评估左滑删除。
- 如果用户看到页面后，会自然把内容理解成“标签集合”“图片墙”“功能卡片区”，通常就不优先用左滑删除。
