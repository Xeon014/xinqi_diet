### [2026-03-11] 首页合并记录页与交互/UI 重构
- 变更范围:
  - 页面/模块: `miniprogram/pages/home/*`, `miniprogram/components/empty-state/*`, `miniprogram/app.json`, `docs/current-ux-issues.md`
  - 后端接口: 无
  - 数据结构: 无
- 做了什么:
  - 将原记录页核心能力并入首页，支持按天切换与日期选择，并按所选日期拉取当日汇总与记录。
  - 调整导航结构，移除“记录”Tab，改为“首页 / 趋势 / 我的”。
  - 首页总览区改为“净热量”主视图，移除“当日状态”标签及多余说明文案，强化核心数值层级。
  - 记录列表改为按“早餐、午餐、晚餐、加餐、运动”分组展示，运动统一聚合；饮食条目标题去除餐次前缀。
  - 删除原“记录加餐”入口，统一为右下角悬浮加号入口，并多轮调整位置与空状态文案。
  - 修复“暂无记录”上方短横线问题：为空状态组件新增 `showIcon` 属性，首页关闭默认占位图标。
  - 同步在 `docs/current-ux-issues.md` 记录“当日状态标签策略”为后期优化项。
- 为什么做:
  - 将高频记录链路集中到首页，减少页面跳转成本。
  - 提升首页首屏信息密度与视觉秩序，让用户更快完成“看状态 + 继续记录”。
  - 收敛空状态与分组展示细节，避免干扰性元素影响主任务。
- 验证方式:
  - 命令: `node scripts/verify-miniprogram-files.js`（涉及小程序改动时已执行并通过）
  - 手动回归页面: 首页（日期切换、净热量总览、分组记录、空状态、右下角加号入口）、饮食编辑页、运动编辑页、趋势页入口可达性
- 风险与回滚点:
  - 风险主要集中在首页交互复杂度上升后的边界状态（跨天切换、空记录日展示）。
  - 如需回滚，可优先恢复 `miniprogram/pages/home/*` 与 `miniprogram/app.json` 到合并前版本。
- 后续待办:
  - 继续优化首页分组记录区的视觉一致性与信息密度。
  - 评估右下角悬浮入口在小屏设备与安全区下的遮挡风险。
