# 客服与助理问答基线

> 目标：为 AI 客服、AI 助理或支持型 agent 提供当前产品事实与标准答法。回答时只基于当前已实现能力，不虚构未来计划，不把历史页面当现有功能。

## 1. 使用原则

- 先回答“当前能做什么”，再给页面入口
- 不承诺未上线能力，不猜测发布时间
- 如果问题涉及异常数据、账号状态、导入失败或明显 bug，先建议用户提供页面、时间、截图或操作步骤
- 如果文档与代码冲突，以当前代码和已验证行为为准，并回写文档

## 2. 产品事实速览

- 产品形态：微信小程序
- 顶层导航：首页 / 趋势 / 我的
- 核心能力：饮食记录、运动记录、健康档案、趋势查看、体重导入、自定义食物、自定义运动、自定义套餐
- 当前趋势页支持：体重、BMI、胸围、腰围、臀围、大腿围
- 当前导入能力：仅支持体重数据导入，不支持围度导入

## 3. 标准问答

| 用户常问 | 标准答法 | 事实来源 |
| --- | --- | --- |
| 这款产品现在能做什么？ | 目前可以记录饮食和运动、查看每日净热量、维护健康档案、查看体重/BMI/围度趋势、管理自定义食物/运动/套餐，也支持导入历史体重数据。 | `docs/面向AI Agent的项目总览.md`、`miniprogram/docs/功能总览清单.md` |
| 怎么记录饮食？ | 可以从首页右下角加号进入食物选择页，选中食物后会在同页底部弹出编辑面板完成保存。也可以直接点首页已有饮食记录进入编辑。 | `miniprogram/README.md` |
| 怎么记录运动？ | 可以从首页右下角加号进入运动选择页，选中运动后在同页底部弹出编辑面板完成保存；已有运动记录也可以直接点进去编辑。 | `miniprogram/README.md` |
| 在哪里看体重趋势？ | 进入底部 Tab 的“趋势”页即可查看体重、BMI 和围度趋势；也可以从“我的 -> 健康档案”底部入口跳过去。 | `miniprogram/README.md`、`miniprogram/docs/页面入口与跳转地图.md` |
| 趋势页现在怎么看？ | 趋势图固定在一个屏幕内展示，点选折线上的日期后，顶部数值会切换到对应点。年和所有时间会展示日级趋势点，横轴只保留开始和结束两个 `YYYY-MM` 标签。 | `docs/面向AI Agent的项目总览.md`、`miniprogram/README.md` |
| 能手动录入 BMI 吗？ | 目前不支持手动录入 BMI。BMI 只支持查看，体重和围度支持新增记录。 | `miniprogram/README.md`、`miniprogram/docs/页面入口与跳转地图.md` |
| 能看单项体征历史吗？ | 可以。在趋势页点某个指标卡右上角的 `···`，会进入该指标的历史明细页。 | `miniprogram/README.md`、`miniprogram/docs/页面入口与跳转地图.md` |
| 能继续补录体征数据吗？ | 可以。体重和围度可以在趋势页快捷记录，也可以在历史明细页继续新增；BMI 仍然不能手工新增。 | `miniprogram/docs/功能总览清单.md` |
| 能导入以前的体重记录吗？ | 可以。从“我的”页进入“导入体重数据”，当前支持 CSV / XLSX 文件，流程是“预览 -> 确认 -> 导入结果”。 | `docs/体重数据导入功能设计.md`、`docs/小程序接口对接说明.md` |
| 导入支持哪些数据？ | 当前只支持体重数据导入，不支持胸围、腰围、臀围等围度导入。 | `docs/体重数据导入功能设计.md` |
| 导入时支持哪些单位？ | 导入会自动识别常见体重单位，当前文档明确支持 `kg / 斤 / lbs`，并统一转换成 kg 入库。 | `docs/体重数据导入功能设计.md` |
| 可以管理自定义食物和运动吗？ | 可以。在“我的”页里进入“自定义食物”或“自定义运动”管理页，支持新建、编辑、删除。 | `miniprogram/README.md`、`miniprogram/docs/功能总览清单.md` |
| 可以管理自定义套餐吗？ | 可以。在“我的”页进入自定义套餐管理页，支持编辑套餐内容、查看整套热量与营养汇总。 | `miniprogram/docs/功能总览清单.md` |
| 第一次使用要注册吗？ | 小程序启动后会自动登录。新用户首次登录会进入资料引导页，可以逐项填写，也可以跳过。 | `docs/小程序接口对接说明.md`、`miniprogram/README.md` |

## 4. 明确不要答错的点

- 不要说“BMI 可以手动录入”
- 不要说“围度支持导入”
- 不要说“趋势图需要左右滑动才能看全”
- 不要说“所有时间还需要手动点加载更早数据”
- 不要把 `pages/food-item-editor` 这类历史页面说成用户当前可见入口

## 5. 碰到异常问题时的建议答法

- 趋势数据看起来不对：
  - 先确认用户查看的是哪个指标、哪个时间区间、点选的是哪一天
  - 再让用户补充截图或操作步骤
- 导入失败或数据异常：
  - 先确认文件格式、日期列、体重列、单位
  - 再收集报错提示或异常行示例
- 登录或新用户引导异常：
  - 先确认当前是新账号还是已有账号
  - 再确认是否刚清过缓存或重新登录
